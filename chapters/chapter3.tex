\section{Results}
\label{sec:results}

The implemented module produces two synchronized output families for each analyzed session:

\begin{enumerate}
  \item \textbf{Continuous authentication outputs:} similarity trajectory, risk score, and confidence level.
  \item \textbf{Behavior learning analytics outputs:} friction points, cognitive load trend, authenticity indicators, and pedagogical recommendations.
\end{enumerate}

From system-level validation, the following operational outcomes are observed:
\begin{itemize}
  \item The event-to-report pipeline remains deterministic for identical input traces, improving reproducibility.
  \item Behavioral markers such as prolonged pauses and deletion bursts are consistently surfaced in friction-point detection.
  \item Sessions with excessive paste behavior receive increased synthetic-risk indicators, improving triage visibility for instructors.
\end{itemize}

%% ── 3.2 Research Findings ────────────────────────────────────
\section{Research Findings}
\label{sec:findings}

The key findings relative to the research objectives are as follows:

\begin{enumerate}
  \item \textbf{Feasibility of integration:} Continuous keystroke authentication and behavior learning analytics can operate within one microservice boundary using shared event inputs.
  \item \textbf{Interpretability gain:} Compared with single authentication scores, combined outputs provide richer context for human decision-making.
  \item \textbf{Pedagogical relevance:} Friction and cognitive-load indicators make assessment telemetry actionable for feedback and remediation.
  \item \textbf{Operational transferability:} API-first design supports integration with assessment workflows and instructor dashboards.
\end{enumerate}

These findings align with prior work showing that keystroke signals are useful for biometric identity \citep{acien2022typenet, senarath2023reevaluating} and that learner trace data is useful for struggle identification \citep{arakawa2021situ, dong2021trace}. The contribution of this dissertation is the practical fusion of both into one deployable module.

%% ── 3.3 Discussion ───────────────────────────────────────────
\section{Discussion}
\label{sec:discussion}

The results highlight an important implementation insight: in educational settings, authentication confidence is more useful when accompanied by behavioral evidence. Purely biometric systems can signal mismatch risk but cannot explain whether observed deviations reflect stress, genuine struggle, device changes, or suspicious external assistance. By pairing identity metrics with process analytics, the module supports proportionate and fairer review decisions.

Compared with large-scale keystroke studies \citep{dhakal2018observations} and deep embedding methods \citep{acien2022typenet}, this work is less focused on benchmark competition and more focused on platform integration, explainability, and classroom applicability. Relative to educational trace-analysis research \citep{arakawa2021situ}, the module extends utility by adding continuous identity confidence, which is critical for high-stakes assessments.

The discussion also surfaces practical constraints. Keystroke-based inference is sensitive to context drift (device changes, keyboard layout, situational stress). Therefore, institutional use should include transparent policies, consent, human-in-the-loop moderation, and threshold calibration by course context.
