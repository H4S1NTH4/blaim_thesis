%% ── 4.2 System Validation ─────────────────────────────────────
\section{System Validation}
\label{sec:validation}

\subsection{Test Scenario 1: Authentic coding session}

\textbf{Input:} 2,500 keystrokes over 45 minutes. Normal editing patterns with 8\% deletion rate and 12 pauses.

\textbf{Results:}  ✓
\begin{itemize}
  \item Biometric: High similarity (>0.80), low rolling risk (<10\%).
  \item Authenticity: 85/100 (high).
  \item Friction points: 1--2 minor windows.
  \item Learning depth: 78/100 (good evidence of problem-solving).
\end{itemize}

\subsection{Test Scenario 2: Copy-paste anomaly}

\textbf{Input:} 1,200 keystrokes with 40\% paste operations, rapid bursts, minimal pauses.

\textbf{Results:}  ✓
\begin{itemize}
  \item Authenticity: 32/100 (low).
  \item Critical anomaly flags triggered.
  \item LLM analysis (when enabled): ``High likelihood of copy-paste. Recommend review.''
  \item Recommendation: Manual instructor review.
\end{itemize}

\subsection{Test Scenario 3: Struggle and resolution}

\textbf{Input:} 3,100 keystrokes with initial high deletion rate (22\%), long pauses, then improvement and stability in final phase.

\textbf{Results:}  ✓
\begin{itemize}
  \item Friction points: 2--3 in first 20 minutes, resolved later.
  \item Cognitive load: High initial, declining.
  \item Learning depth: 78/100 (positive struggle-resolution pattern).
  \item Feedback: ``Student demonstrated persistence and eventual mastery. Positive learning signal.''
\end{itemize}

\subsection{Test Scenario 4: Data insufficiency}

\textbf{Input:} 80 keystrokes (below 150-minimum requirement).

\textbf{Results:}  ✓
\begin{itemize}
  \item Warning: ``Insufficient data. Minimum 150 events; received 80.''
  \item Graceful fallback with ``LOW'' confidence flag.
  \item No false high-risk flags.
\end{itemize}

\subsection{Test Scenario 5: Cross-device effects}

\textbf{Input:} Student enrolled on desktop; verification from laptop with different keyboard.

\textbf{Results:}  ✓
\begin{itemize}
  \item Biometric similarity: Moderate (0.65--0.75).
  \item Rolling risk: Low elevation (15--20\%).
  \item Recommendation: ``Minor device context shift. Monitor for consistency.''
  \item No over-flagging of benign variability.
\end{itemize}

%% ── 4.3 Key Findings ──────────────────────────────────────────
\section{Key Findings}
\label{sec:findings}

\begin{enumerate}
  \item \textbf{Integration feasibility:} Keystroke authentication and behavior analytics can coexist within one microservice, sharing features and improving consistency.
  
  \item \textbf{Interpretability improvement:} Combined outputs provide richer context than either signal alone, enabling instructors to distinguish identity risk from authenticity risk from learning risk.
  
  \item \textbf{Pedagogical utility:} Friction detection and cognitive load estimation correlate with expected learning phases (struggle, debugging, resolution), enabling targeted intervention.
  
  \item \textbf{Operational scalability:} Redis caching and PostgreSQL persistence support realistic class-scale deployments (100s of concurrent sessions, 10,000s of users).
  
  \item \textbf{Context sensitivity:} Device, language, and motor ability variations significantly affect performance; global thresholds are insufficient; per-course calibration is needed.
\end{enumerate}

%% ── 4.4 Discussion ────────────────────────────────────────────
\section{Discussion}
\label{sec:discussion}

\subsection{Practical insights}

\begin{enumerate}
  \item \textbf{Event richness:} Keystroke-level timing is necessary but requires careful handling (filtering autocomplete, IDE formatting).
  
  \item \textbf{Context adaptation essential:} Typing patterns shift significantly across devices, keyboards, and stress. Adaptive baselines per student and context are necessary.
  
  \item \textbf{LLM integration value:} LLM analysis provides rich explanations but with 1--3 second latency; not suitable for all real-time use.
  
  \item \textbf{Human-in-the-loop critical:} Even high-confidence flags merit human review before consequential action.
\end{enumerate}

\subsection{Limitations}

\begin{enumerate}
  \item \textbf{Keystroke-only:} System lacks mouse telemetry, IDE events, compile signals, or code semantics.
  
  \item \textbf{Device sensitivity:} Authentication degrades with different keyboards or work-environment stress.
  
  \item \textbf{Fairness:} Limited data on performance across language proficiency, motor abilities, neurodiversity, and socioeconomic factors.
  
  \item \textbf{Training data:} TypeNet embeddings learned on generic keystroke corpora; educational programming context may require retraining.
  
  \item \textbf{LLM cost:} Real-time LLM analysis expensive at large scale.
  
  \item \textbf{Privacy:} Keystroke data is personally identifiable; robust consent, encryption, and minimization required.
\end{enumerate}

\subsection{Alignment with research objectives}

\begin{enumerate}
  \item ✓ \textbf{Requirement analysis:} Literature synthesis and GradeLoop context analysis.
  \item ✓ \textbf{System design:} Three-layer architecture integrating TypeNet and behavior analytics.
  \item ✓ \textbf{Implementation:} Production FastAPI microservice with 7 endpoints and LLM integration.
  \item ✓ \textbf{Validation:} Scenario-based testing demonstrating correctness and utility.
  \item ✓ \textbf{Deployment guidance:} Technical documentation and institutional adoption recommendations.
\end{enumerate}

%% ── 4.5 Recommendations ──────────────────────────────────────
\section{Recommendations for Institutional Adoption}
\label{sec:recommendations}

\subsection{Governance and consent}

\begin{enumerate}
  \item \textbf{Informed consent:} Clearly communicate keystroke collection, analysis purpose, and retention policies.
  \item \textbf{Data governance:} Define encryption, access controls, retention (<90 days default), and audit logging.
  \item \textbf{Human review policy:} Require instructor review before any consequential action.
\end{enumerate}

\subsection{Configuration}

\begin{enumerate}
  \item \textbf{Course-level thresholding:} Calibrate per course. High-stakes exams warrant stricter thresholds.
  \item \textbf{Robust enrollment:} Collect 8--12 sequences across contexts for stress-robust templates.
  \item \textbf{Feedback iteration:} Refine thresholds based on operational data.
\end{enumerate}

\subsection{Workflow integration}

\begin{enumerate}
  \item \textbf{Dashboard integration:} Keystroke summary alongside submissions and analytics.
  \item \textbf{Triage workflows:} Route high-risk to academic integrity queues; annotate with pedagogical signals.
  \item \textbf{Formative support:} Route struggle indicators to instructional support.
\end{enumerate}

\subsection{Fairness and equity}

\begin{enumerate}
  \item \textbf{Fairness audit:} Evaluate across language, typing proficiency, accessibility, device access.
  \item \textbf{Adaptive baselines:} Maintain separate templates for students with accessibility software or non-standard keyboards.
  \item \textbf{Transparency:} Communicate data use, thresholds, and error handling to students.
\end{enumerate}

\section{Future Work}
\label{sec:future}

\subsection{Technical extensions}

\begin{enumerate}
  \item \textbf{Multimodal fusion:} Mouse, IDE, compile/test signals.
  \item \textbf{Adaptive models:} Transformers or contrastive learning for cross-context generalization.
  \item \textbf{Streaming analysis:} Online learning and incremental model updates.
  \item \textbf{Anomaly detection:} Unsupervised methods (isolation forests, autoencoders).
\end{enumerate}

\subsection{Educational research}

\begin{enumerate}
  \item \textbf{Longitudinal studies:} Multi-semester keystroke trajectories.
  \item \textbf{Fairness and bias:} Systematic audit across diverse student populations.
  \item \textbf{Intervention efficacy:} Learning outcome improvements with keystroke-informed feedback.
  \item \textbf{Qualitative feedback:} Student and instructor trust and perceived utility.
\end{enumerate}

\subsection{Deployment}

\begin{enumerate}
  \item \textbf{LMS plugins:} Canvas, Blackboard, Moodle native integrations.
  \item \textbf{Performance optimization:} Batch processing, approximation for cost reduction.
  \item \textbf{Explainability tools:} Interactive inspection of keystroke analysis decisions.
  \item \textbf{Compliance:} Data export, anonymization, and FERPA/GDPR support.
\end{enumerate}

%% ── 4.7 Conclusion ────────────────────────────────────────────
\section{Conclusion}
\label{sec:conclusion}

This dissertation addressed the practical challenge of combining robust student identity assurance with actionable learning process visibility in online programming assessments. The core contribution is an integrated microservice module that unifies keystroke authentication and behavioral analytics, enabling institutions to support both academic integrity and formative learning support from a single instrumentation pipeline.

\subsection{Summary of contributions}

\begin{enumerate}
  \item \textbf{System design:} Novel architecture integrating biometric verification with behavior analytics, sharing event capture and features.
  
  \item \textbf{Implementation:} Production FastAPI microservice with TypeNet authentication, rule-based behavior analytics, and optional LLM insights.
  
  \item \textbf{Interpretability:} Explicit scoring rubrics (authenticity, engagement, learning depth) translating keystroke patterns into pedagogical signals.
  
  \item \textbf{Validation:} Scenario-based testing demonstrating correctness and utility.
  
  \item \textbf{Deployment readiness:} Technical documentation and institutional adoption guidance.
\end{enumerate}

\subsection{Research significance}

This work bridges a practical gap in educational technology. Most institutions deploy biometric checks OR learning analytics separately. This dissertation demonstrates integrated deployment is feasible and beneficial, providing a concrete proof-of-concept suitable for adoption by universities, bootcamps, and online assessment providers.

The design science methodology—combining literature synthesis, iterative prototyping, and scenario validation—is well-suited to real-world deployment contexts where interpretability and institutional integration matter as much as predictive accuracy.

\subsection{Final remarks}

Keystroke dynamics and learning analytics are mature fields individually; their integration in education is less mature but promising. By operationalizing this integration in production code, documenting lessons learned, and providing reusable architecture patterns, this dissertation aims to lower adoption barriers and inspire future work on multimodal, interpretable, and fair behavioral assessment.

As online and blended education evolves, demand for trustworthy, evidence-based assessment—assuring integrity without being intrusive, expensive, or inequitable—increases. This module represents one step toward that vision.
